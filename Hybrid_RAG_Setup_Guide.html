Hybrid RAG Pipeline (BM25 + BGE-M3 + Qwen2.5:7b)

Overview: We successfully transitioned the chatbot from relying solely on Gemini to a local/hybrid AI model using Qwen2.5 (7 Billion Parameters) via Ollama, combined with a hybrid retrieval strategy consisting of BM25 (Sparse/Lexical Keyword Match) and BAAI BGE-M3 (Dense Semantic Match).

What Was Built / Changed

File Purpose
bm25_rag/hybrid_bge_qwen_retriever.py Hybrid Retriever: Uses Reciprocal Rank Fusion (RRF) to merge the top results of BM25 (keyword matching) and BGE-M3 (ChromaDB semantic matching). Also includes a fallback to BM25 if the BGE vector store fails.
bm25_rag/hybrid_bge_qwen_generator.py Local Generator: Connects to your local Ollama server to feed the retrieved documents to the qwen2.5:7b model and streams back an Arabic legal response.
backend/app.py New Endpoint: Mapped POST /chat/hybrid to execute the hybrid retriever and generator. Added a health check for it at GET /health/hybrid. Previously existing /chat with Gemini is left strictly isolated.
frontend/app/(tabs)/chat.tsx Frontend Wiring: Configured BACKEND_URL to point to your physical network IP (192.168.1.9), ensuring mobile devices on the same WiFi can reach the server. The send button and voice recorder now point to /chat/hybrid.

How to Run Your System From Scratch

Step 1: Start Ollama

You need your local LLM engine running and the specific Qwen model downloaded to your system.

# Open a command prompt / PowerShell and run:
ollama serve

# If you haven't pulled the model recently, run in a separate tab:
ollama pull qwen2.5:7b

Step 2: Ensure Data & Indices exist

For the hybrid setup to work, both retrieval databases must be successfully generated beforehand.

  1. BM25 Corpus: Generates the bm25_corpus.pkl and index files. 
    python -m bm25_rag.bm25_indexer
  2. BGE-M3 Embeddings: Connects to ChromaDB. 
    python rag_bge/bge_embed_articles.py

Step 3: Boot Up the FastAPI Backend

Starts the API. We bind it to 0.0.0.0 so phones/emulators on the same WiFi network can access it via your PC's IP address.

cd "C:\Users\pc cam\Desktop\L3\pfe baya\Istacherni"
uvicorn backend.app:app --host 0.0.0.0 --port 8000 --reload

Verify it is running properly by visiting: http://192.168.1.9:8000/health/hybrid in your browser.

Step 4: Start Expo (Frontend)

Launch your React Native App.

cd "C:\Users\pc cam\Desktop\L3\pfe baya\Istacherni\frontend"
npx expo start -c

Note: The -c clears the metro cache which is helpful after IP address changes in chat.tsx.

Important Note on IP Addresses:

In frontend/app/(tabs)/chat.tsx, the BACKEND_URL is hardcoded to http://192.168.1.9:8000. If your router assigns you a new network IP (e.g. 192.168.1.15), you must edit chat.tsx and restart Expo. To check your IP, run ipconfig in your terminal.
