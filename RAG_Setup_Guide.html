Architecture Overview

The RAG pipeline works in two phases:

PhaseWhenWhat Happens
Ingestion Once (offline) Load all JSON law articles → Chunk them → Embed with Gemini → Store in ChromaDB
Query Every user question Embed the question → Search ChromaDB → Retrieve top-5 articles → Generate Arabic answer with Gemini

File structure:

Istacherni/
├── .env                                   ← API key (you create this)
├── requirements.txt                       ← Python packages
├── test_rag.py                            ← Quick test script
├── dataset/raw/**/*.json                  ← Law data (already exists)
└── backend/
    ├── app.py                             ← FastAPI server  POST /chat
    ├── vectorstore/                       ← ChromaDB database (auto-created)
    └── rag/
        ├── preprocessing/normalize_arabic.py
        ├── ingestion/load_articles.py
        ├── chunking/chunk_articles.py
        ├── embedding/embed_articles.py    ← Run once to build vector store
        ├── retrieval/retriever.py
        └── generation/generator.py

Prerequisites — do this once

Open PowerShell and navigate to your project folder:

cd "c:\Users\pc cam\Desktop\L3\pfe baya\Istacherni"

Verify Python is installed:

python --version
# Expected: Python 3.11+ or 3.13

Install all required packages:

pip install -r requirements.txt
✅ Wait for the install to finish. You should see "Successfully installed..." at the end. This installs: google-genai, chromadb, fastapi, uvicorn, python-dotenv.

API Key Setup — do this once

The chatbot uses Google's Gemini AI. You need a free API key.

1. Go to: https://aistudio.google.com/app/apikey

2. Click "Create API Key" and copy it.

3. Create a file named .env in the project root:

GEMINI_API_KEY=your_actual_key_here

4. Verify it works:

python -c "from dotenv import load_dotenv; load_dotenv(); import os; print('Key OK:', bool(os.getenv('GEMINI_API_KEY')))"
# Expected: Key OK: True
⚠️ Never share or commit your .env file. It is already listed in .gitignore.

ℹ️ If generation shows 429 rate-limited, waiting… — the script handles it automatically and retries. If it fails after 5 retries, your free API key has hit its daily generation limit. Wait until the next day and try again.

Call the Chatbot API

Health check:

Invoke-WebRequest -Uri "http://localhost:8000/health" -Method GET |
  Select-Object -ExpandProperty Content

Expected: {"status":"ok","indexed_articles":777,...}

Ask a legal question (PowerShell):

$body = '{"question": "ما هي عقوبة السرقة في القانون الجزائري؟"}'

Invoke-WebRequest -Uri "http://localhost:8000/chat" `
  -Method POST `
  -ContentType "application/json" `
  -Body $body | Select-Object -ExpandProperty Content

Response format:

{
  "question": "ما هي عقوبة السرقة في القانون الجزائري؟",
  "answer": "وفقاً لقانون العقوبات الجزائري، تنص المادة 350 على أن السرقة...",
  "sources": [
    {
      "id": "DZ_PENAL_ART_350",
      "law_name": "قانون العقوبات",
      "law_domain": "Penal Code",
      "article_number": "350",
      "title": "تعريف السرقة",
      "text_original": "المادة 350: ...",
      "score": 0.2747
    }
  ]
}

Control the number of source articles returned:

$body = '{"question": "ما هي شروط عقد البيع؟", "top_k": 10}'
# top_k: 1 to 20 (default: 5)

Troubleshooting

Error Cause Fix
uvicorn not found uvicorn not on PATH Use python -m uvicorn ...
GEMINI_API_KEY not set No .env file Create .env with your key
Vector store not found Step 3 not run yet Run embed_articles.py first
429 RESOURCE_EXHAUSTED (embedding) Per-minute quota hit Script retries automatically — wait
429 RESOURCE_EXHAUSTED (generation) Daily free limit hit Wait until tomorrow OR change model to gemini-1.5-flash
Ingestion stuck / no progress Waiting on rate-limit retry Normal — do not interrupt. Check terminal for [WARN] waiting Xs...
ModuleNotFoundError Wrong working directory Always run from the project root: cd "…\Istacherni"

Switch generation model (if hitting daily limits):

In backend/rag/generation/generator.py line 30, change:

GENERATION_MODEL = "gemini-1.5-flash"   # separate free quota from 2.0-flash
