Istacherni RAG Run Guide

Step-by-step instructions to run the BM25, Graph, and Agentic RAG pipelines for the Algerian Law chatbot.

BM25 RAG Graph RAG Agentic RAG
⚙️
Before You Start

Common Prerequisites

These steps apply to all three RAG models. Complete them once before running any pipeline.

1

Open a Terminal in the Project Root

Navigate to the Istacherni project folder.

# On Windows (PowerShell)
cd "c:\Users\pc cam\Desktop\L3\pfe baya\Istacherni"
2

Activate the Virtual Environment

Always activate .venv first so the correct packages are used.

.venv\Scripts\activate
# Your prompt should show (.venv) when activated
3

Install All Dependencies (First Time Only)

pip install rank-bm25 networkx numpy python-dotenv google-generativeai
ℹ️ rank-bm25 is required by BM25 RAG and Agentic RAG. networkx is required by Graph RAG only.
4

Verify the .env File

Make sure the following keys are present in .env at the project root:

BM25_GEMINI_API_KEY Used by BM25 generator
GRAPH_GEMINI_API_KEY Used by Graph generator
AGENTIC_GEMINI_API_KEY Used by Agentic agent
GEMINI_API_KEY Standard RAG key
🔍
Pipeline 1

BM25 RAG

Uses BM25Okapi keyword ranking to retrieve law articles, then generates an Arabic answer with Gemini. Retrieval is offline — only generation uses the API.

1

Build the BM25 Index (Run Once)

Reads all law articles from dataset/raw/, tokenizes them with the Arabic tokenizer, and saves the index to disk.

python bm25_rag/bm25_indexer.py

Expected output:

[BM25-Indexer] Loading articles… [BM25-Indexer] Tokenising corpus… [BM25-Indexer] Building BM25Okapi over XXXX docs… [BM25-Indexer] ✓ Index saved → bm25_rag/index/bm25_index.pkl [BM25-Indexer] ✓ Corpus saved → bm25_rag/index/bm25_corpus.pkl Corpus size : XXXX articles Vocab size : XXXXX terms
✅ Two files are created: bm25_rag/index/bm25_index.pkl and bm25_rag/index/bm25_corpus.pkl. Next time you run the test, they are loaded automatically.
2

Run the End-to-End Test

Retrieves top-5 articles for 3 Arabic queries and generates answers using Gemini.

python test_bm25_rag.py

Expected output:

================================================================ BM25 RAG — End-to-End Test ================================================================ ================================================================ QUERY: ما هي عقوبة السرقة في القانون الجزائري؟ ================================================================ 🔎 Step 1 — BM25 Retrieval Retrieved 5 articles: ├─ [القانون...] المادة 350 score=12.3456 | ... 🤖 Step 2 — Answer Generation (Gemini 2.5 Flash) ✅ Answer: ------------------------------------------------------------ ...Arabic answer... ------------------------------------------------------------ 🏁 Test complete.
3

Optional: Skip Generation (No API Calls)

Open test_bm25_rag.py and set line 25:

USE_GENERATION = False   # Change from True to False

Then re-run the test — it will only test BM25 retrieval without using the Gemini API.

Pipeline 2

Graph RAG

Builds a Knowledge Graph (NetworkX DiGraph) from law metadata — no API needed for the graph. Retrieval uses graph traversal (keyword → article → related article) ranked by PageRank + hit count.

1

Build the Knowledge Graph (Run Once)

Builds the graph from JSON metadata in dataset/raw/. No API calls needed.

python -m graph_rag.graph_builder

Expected output:

[GraphBuilder] Loading articles from dataset… [GraphBuilder] XXXX articles loaded. [GraphBuilder] Phase 1 — Adding nodes… [GraphBuilder] Phase 2 — Adding edges… [GraphBuilder] Graph complete: Nodes : XXXXX (XXXX articles + concept/domain/penalty nodes) Edges : XXXXXXXX [GraphBuilder] ✓ Saved → graph_rag/cache/law_graph.pkl ── Node type counts ── article : XXXX concept : XXXX domain : XX penalty : X ── Top 10 most connected article nodes ── degree=XXXX ART_XX [...]
✅ Two cache files are created: graph_rag/cache/law_graph.pkl and graph_rag/cache/graph_corpus.pkl. They are reloaded automatically on future runs.
2

Run the End-to-End Test

Traverses the knowledge graph for 3 Arabic queries and generates answers with Gemini.

python test_graph_rag.py

Expected output:

==================================================================== GRAPH RAG — End-to-End Test ==================================================================== ==================================================================== QUERY : ما هي عقوبة السرقة في القانون الجزائري؟ ==================================================================== 🕸️ Step 1 — Knowledge Graph Traversal Retrieved 5 articles: ├─ [القانون...] المادة 350 graph=0.854 pr=0.000123 | ... 🤖 Step 2 — Answer Generation (Gemini 2.5 Flash) ✅ Answer: ────────────────────────────────────────────────────────────── ...Arabic answer... ────────────────────────────────────────────────────────────── 🏁 All tests complete.
3

Optional: Skip Generation (No API Calls)

Open test_graph_rag.py and set line 25:

USE_GENERATION = False   # Change from True to False

This tests only the graph traversal and ranking without any Gemini calls.

4

How Graph Retrieval Works

Understanding the retrieval logic:

🔑 HAS_KEYWORD Article → Concept (keyword)
🔗 RELATED_TO Article → Article (related)
📂 IN_DOMAIN Article → Law Domain
⚖️ HAS_PENALTY Article → Penalty Class
🤖
Pipeline 3

Agentic RAG

A Gemini-powered agent that autonomously decides which search tools to call, may search multiple times (broad → domain-specific), and produces a richer, self-consistent Arabic answer.

1

No Manual Index Build Needed

The Agentic RAG builds its BM25 index automatically the first time it runs. The index is cached in agentic_rag/cache/.

ℹ️ The agent's 3 built-in tools: search_articles(query, top_k), filter_by_domain(domain, query, top_k), get_article_by_id(article_id)
2

Run the End-to-End Test

The agent runs autonomously, deciding how many search rounds are needed for each query.

python test_agentic_rag.py

Expected output (first run builds the cache):

[AgentKB] Building index from dataset… [AgentKB] Index built: XXXX docs, XXXXX terms. ==================================================================== AGENTIC RAG — End-to-End Test ==================================================================== ==================================================================== QUERY : ما هي عقوبة السرقة في القانون الجزائري؟ ==================================================================== 📊 Agent used 2 round(s), called 3 tool(s): ├─ Round 1 | search_articles({'query': '...', 'top_k': 5}) → X results ├─ Round 1 | filter_by_domain({'domain': 'جنائي', ...}) → X results ├─ Round 2 | get_article_by_id({'id': 'ART_350'}) → found ✅ Final Answer: ──────────────────────────────────────────────────────────── ...Rich Arabic answer... ──────────────────────────────────────────────────────────── 🏁 All tests complete.
⚠️ Agentic RAG makes multiple Gemini API calls per query (one per reasoning round). This consumes more quota than BM25 or Graph RAG.
3

Optional: Disable Verbose Reasoning

Open test_agentic_rag.py and set line 29:

VERBOSE = False   # Hides the agent's internal reasoning steps
4

Agent Architecture

🧠 Multi-Round Reasoning Agent may loop across multiple search rounds
🔧 Tool Autonomy Selects tools independently per query
📊 BM25 Backend Underlying search still uses BM25Okapi
⚡ Auto Cache Index built once, reloaded on next runs
